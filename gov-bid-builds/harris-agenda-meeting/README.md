## What this is

A web app pilot we ship alongside the proposal. The pilot is a working demonstration of the system the agency is asking for — they can click a link in our proposal and see real screens with seeded fake data flowing through them.

The proposal narrative references this demo URL repeatedly. The demo is the bid's differentiator.

## Two-week sprint goal

Build a working pilot that supports the agency's demo storyline (described in [requirements.md](./requirements.md)) and deploys to a public URL we can paste into the proposal.

## Quickstart

```bash

# fill in the env vars
```

## Demo storyline

The agency demo flow is in the build BC list. Each task is one screen / one feature in the flow. Build top-down.

## Fake-data seeds

```bash
npm run seed
```

The seed script populates the demo with realistic sample data so visitors see a working system, not empty tables. Seed templates live in [seeds/](./seeds/).

## Deployment

```bash
npm run deploy
```

Default target: Hetzner VPS at `95.216.199.47` with nginx reverse proxy. See [deploy/](./deploy/) for the Dockerfile + compose config.
