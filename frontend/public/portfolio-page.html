Tier-A · structured

Build artifacts

One fixed artifact slot per week of the 12-week Lego model. Because every student fills the same twelve slots, Tier-A is gradeable and directly comparable — the spine of the readiness score's Build pillar.
A slot becomes Verified only when its linked requirement is satisfied by a real GitHub commit and graded by a mentor. Slots unlock on the cohort's just-in-time drip — a locked slot isn't "missing," it just isn't due yet.

Tier-B · flexible

Showcase artifacts

The flexible, employer-facing assets each student assembles for their project. The system scaffolds the slot and AI-drafts a first version from the build evidence; the student edits and publishes. Four canonical types, plus room to add more.
AI drafting: the explainer script, one-pager copy, and deck outline are generated from the student's requirements, commits, and build-log — then handed to the student to refine. Drafting never publishes on its own; the student always approves.

Readiness score

A single 0–100 signal of how job-ready the student's work is — composed from three deterministic pillars, recomputed from evidence on every push, and fully retunable in config (no deploy).

Why it's defensible: every input is evidence-backed (a verified commit, a published asset, a real course completion). The score can't be inflated by self-report, and it recomputes to the same number from the same evidence — so two students with the same readiness genuinely did comparable work.

Build spec

The plug-in layer behind this mockup: data model, the readiness formula, public/private endpoints, per-student isolation, idempotency, and failure-first render rules. Maps to BUILD_SPEC §6, §7, and EPIC 5.
Data model extends §7 — reuses portfolioGenerationService.ts
Project (readiness_score, github_repo, preview_url, public_slug, is_public) ├── Requirement (state: unmatched→matched→partial→verified, github_evidence_sha) ├── Artifact │ tier: 'A' | 'B' │ slot: 'week_03_architecture' | 'showcase_demo_video' … │ status: locked | in_progress | in_review | verified | published | draft │ grade: null | 'A' | 'B' … // Tier-A only, mentor-set, PRIVATE │ url, ai_drafted:boolean, published_at └── BuildLog (weekly; feeds the AI narrative + strengths ranking)
Readiness formula
PillarWeightComputed from
build0.50verified_tierA / total_tierA_due × 100 (only slots whose week has unlocked)
showcase0.25weighted published Tier-B slots (demo+deck weighted higher than one-pager)
course0.25mirrored Skilljar course progress for the cohort's due weeks
Deterministic & idempotent: readiness = f(evidence) — pure function of verified requirements, published artifacts, and mirrored progress. Recompute on every GitHub webhook; same evidence → same score, no drift. Weights in config/readiness.json so Ali/Aleem retune without a deploy.
Endpoints
RouteAuthReturns

Per-student isolation & failure-first
ConcernRule
IsolationPublic endpoint resolves a slug → one project and selects only public-projection columns. No join can reach another student's drafts, grades, email, or ledger. Every authed read filters on enrollment_id from the session.
Private fieldsMentor grades & feedback, email, points ledger, peers, and unpublished drafts are excluded at the query layer — not hidden in CSS.
GitHub sync downReadiness renders from the last persisted score with a "as of <time>" stamp; never blocks the page. Webhook retries with capped backoff; failures dead-letter for manual replay.
AI draft fails / times outSlot falls back to the empty "draft a first version" state; the student can retry. A failed draft never publishes and never overwrites an existing published asset.
Missing artifactLocked/empty slots render as first-class states (see States & variants) — an incomplete portfolio is a valid portfolio, not an error.
IdempotencyScore recompute, AI draft, and publish are all keyed so a replayed webhook or double-click can't double-write or double-post.
Tests Kes must ship (per CLAUDE.md): happy (verified slot renders + scores), failure (GitHub/AI down → fallback render), boundary (0 artifacts, all 12 verified, slug not found → 404 not 500), and isolation (student A's slug never leaks student B's private fields).
