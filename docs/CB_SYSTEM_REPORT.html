CB System - Architecture & Management Report

The autonomous Basecamp agent: every moving part, how the pieces connect, what data it produces, and how you manage it. Prepared for Ali Muwwakkil.
Contents: 1 · Engine vs product 2 · The layers 3 · @CB mention lifecycle 4 · Data it produces 5 · The dashboard 6 · Reliability work

CB System is the autonomous agent that lives inside Basecamp. It reads @-mentions, drafts deliverables, scores and tracks to-dos, watches itself, and reports to you. It is the engine that the advisor app's /my-day product is built on - it is what connects the two projects.

3What happens when you @-mention CB

sequenceDiagram
  participant Ali
  participant BC as Basecamp
  participant DSP as Dispatcher (3-min)
  participant CTX as Context walker
  participant LLM as GPT-4o handler
  participant LOG as Audit log
  Ali->>BC: comment "@CB System draft the reply…"
  DSP->>BC: poll new comments
  BC-->>DSP: new @CB mention
  DSP->>DSP: dedup + allowed-requester + not-an-agent-card
  DSP->>CTX: gather thread context
  CTX->>BC: todolist + todo + comments + linked docs
  BC-->>CTX: full context
  CTX-->>LLM: context bundle
  LLM->>LLM: decide + draft (breaker: max 2 replies/comment)
  LLM->>BC: post reply (draft) / queue follow-up
  LLM->>LOG: record invocation (who · tools · outcome)

The circuit breaker caps replies per comment so a bug can never spam a thread again. The audit log (cb-handler-log.jsonl) powers the dashboard's Activity feed.

5How you manage it - the dashboard

enterprise.colaberry.ai/admin/cb-system - six panes, fed entirely by the data above (no new instrumentation):

  1. Health - one GREEN / YELLOW / RED light: is it running, catching mentions, erroring, or tripping its safety breaker.
  2. Throughput & cost - chart of requests handled and agent cost over time.
  3. Per-project - which projects it's working, open to-dos, mentions answered.
  4. Activity feed - live list of what CB just did (who asked, what, outcome).
  5. Exceptions & quality - errors and quality flags; the "needs a human look" list.
  6. Subsystems - each component's status + a "Run sync now" control.
The mirror that the priority engine reads now tracks only real work (~1.6k to-dos) after excluding the high-volume Center of Excellence (student) and RMG Mortgage (bulk) projects.
