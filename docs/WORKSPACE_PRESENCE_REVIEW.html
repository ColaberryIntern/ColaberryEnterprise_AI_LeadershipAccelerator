2RecentlyMovedCard — operational history feeling

New card slot directly below the priority card. Hidden when nothing has moved — that's intentional. Renders only when at least one of readiness / coverage / queue / health has changed since the last snapshot.

Why you don't see it in the screenshot above: first capture after deploy — no prior snapshot to compare against. The card appears on second visit (or any time state changes between visits).

What renders when there's momentum:
  • Eyebrow + relative-time strap: Recently moved · Last visit 23 minutes ago
  • One delta chip per moved dimension. Forward = green ↑ chip, backward = red ↓ chip
  • Each chip carries an arrow, the signed delta, and a one-line caption explaining what that direction means ("more requirements matched" / "regression(s) since last visit" / etc.)
  • Subtle ws-delta-rise entrance — 360ms opacity + 4px translate. No looping.

4Grouped progression + 30s toast suppression

The MicroToast host previously fired one toast per detected delta. If readiness AND coverage both improved in the same poll, two toasts queued sequentially (≈7s of toast time). That's noisy. Now:

What changed in MicroToast:
  • Grouped progression. Readiness ≥+3 and Coverage ≥+3 in the same poll → one combined toast: "Forward motion: Readiness +N, Coverage +M" with a graph-up-arrow icon. Suppresses the two solo toasts.
  • 30-second signature dedup. Each toast carries a signature (readiness-up, next:src_123, build-start:foo). Re-firing the same signature within 30s is a no-op. Prevents poll-jitter from re-flashing the same toast.
  • New coverage solo toast. Coverage ≥+3 (without readiness improving) now fires its own toast ("Coverage +N% (now M%)"). Previously only readiness was watched.
  • Still ≤1 toast on screen. Queue head drains sequentially. Calm.

5Ambient motion primitives — ws-breath / ws-fresh / ws-pulse-dot / ws-shimmer / ws-delta-rise

Rules baked into the file:
  • No animation faster than 2.5s on a static element

  • prefers-reduced-motion: reduce shuts every class down — content remains, motion gone

6Contextual surface memory + drawer continuity

useWorkspaceMemory grew from 4 fields to 11. The workspace now remembers more about what you were doing across reloads + tab switches. Memory is in localStorage (cross-tab synced); reduced to in-memory if blocked.

New fields:
  • lastDrawerOpen — last drawer the operator opened on Home
  • lastSystemTab — last System View tab (reserved for next sprint)
  • lastBpId — last BP detail viewed (reserved)
  • lastReadinessScore / lastCoverageScore / lastQueueSize / lastHealthScore — snapshot values for delta computation
  • lastBuiltAt — the state.built_at at last snapshot, prevents redundant writes
  • lastSnapshotAt — drives "Last visit X ago" string
New helper: recordSnapshot()

Called by CoryHome on every state poll. No-ops when state.built_at is unchanged — so it writes localStorage at most once per actual rebuild (not once per render).

What's NOT yet wired

lastDrawerOpen is captured but no "restore last drawer" affordance ships in this sprint — the affordance design (subtle "Open last: Coverage" link on the page?) wants its own review. lastSystemTab + lastBpId are reserved fields not yet read by System View — that's the next sprint's pickup.
