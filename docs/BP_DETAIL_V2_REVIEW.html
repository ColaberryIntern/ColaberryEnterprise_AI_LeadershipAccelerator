2Two BPs, different shapes

The "Where it stands" two-column section adapts to each BP — proves the new layout is not just chrome.

Dataset Registration · "In place": Frontend surface (forming) + Agent / autonomous layer · "Still needed": Backend implementation
Lead Management · "In place": Backend (forming) + Frontend (forming) + Agent · "Still needed": "Nothing flagged as missing — the layer set is complete."
How "In place" / "Still needed" is derived (no new endpoints):
  • usability.backend === 'ready' → "Backend implementation" in "In place"
  • usability.backend === 'partial' → "Backend implementation (forming)" in "In place"
  • usability.backend === 'missing' (or absent) → "Backend implementation" in "Still needed"
  • Same rules for frontend ("Frontend surface") and agent ("Agent / autonomous layer")
  • Open gaps count appended to "Still needed" as one line ("3 open gaps flagged by analysis")
  • Both columns have empty-state copy so the surface never shows a bare list

3Maturity progression — calm, not numeric

The old modal had "Maturity: L1 Prototype → L5 Target" as a numbered section #7. The new strip shows the journey at a glance.

Mapping

LevelLabelBlurb shipped
L1Prototype"Early structure exists; rough edges are expected."
L2Working"Core path runs; edges and recovery paths still informal."
L3Reliable"Documented happy path; supporting infrastructure in place."
L4Hardened"Verified end-to-end; observability is in the loop."
L5Mature"Steady state; ongoing work is refinement."
Visual rules:
  • 5 dots at 14×14 in a horizontal strip, separated by 16×2 connectors
  • Filled dots = at or below current level (primary blue); empty dots = above current
  • Connector between two filled dots is also primary blue; otherwise gray
  • Sits in a calm bg-alt container — no white card-on-card stacking
  • 200ms ease transition on dot color so a level promotion feels intentional
  • "Not yet rated" empty state when maturity.level is 0 or absent

4Next steps — same prompts, calmer presentation

The old modal had 3 colored "Fix Backend / Add UI / Enhance Agent" buttons in a toolbar at the top + a Learn button + a USABLE/NOT READY badge. The new "Next steps" section at the bottom carries three quiet outline buttons with explanatory hover titles.

  • Outline buttons (white background, primary text, light border) — no colored fills
  • Tooltip on hover explains what the button drafts ("Drafts a Claude Code prompt to extend the backend layer of this BP")
