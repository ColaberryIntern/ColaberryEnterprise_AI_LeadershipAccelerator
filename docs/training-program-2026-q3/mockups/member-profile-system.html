The ring fills with progress toward the next tier. The tile colour is the tier colour; the Roman numeral is the tier rank. sm rides in the nav + leaderboard rows, md on cards, lg on the profile hero, xl on the level screen.
How leveling works
TierNameReach atWhat it signals
Deterministic & idempotent: a member's level is a pure function of their cumulative points — level = max(tier where points ≥ tier.min). Recompute any time from the points ledger; it never drifts. Thresholds live in config, not code, so Aleem/Ali can retune without a deploy.

1 · The Member entity (extends §7 data model)
// backend/src/models/Member.ts — the gamification + profile contract interface Member { id: string; // uuid enrollmentId: string; // FK → Enrollment (cohort + tenant boundary) cohortId: string; // scopes the directory + leaderboard displayName: string; headline: string; // "AI Systems Architect in training" bio: string; avatarUrl: string | null; // null → initials fallback role: 'student' | 'mentor' | 'admin'; skills: string[]; links: { kind: 'github'|'linkedin'|'portfolio'|'x'; url: string }[]; // --- gamification (deterministic, derived from the points ledger) --- points: number; // cumulative; sum(PointEvent.delta) level: 1|2|3|4; // derived: levelFor(points) streakDays: number; contributions: { posts: number; comments: number; likesGiven: number; labsShipped: number; artifacts: number }; presence: 'online' | 'idle' | 'offline'; // P0: poll/heartbeat optOutLeaderboard: boolean; // privacy → renders as "—" joinedAt: string; // ISO-8601 } // PointEvent is the source of truth; points/level/contributions are projections. interface PointEvent { id: string; memberId: string; cohortId: string; kind: PointKind; delta: number; refId: string; // refId = idempotency key createdAt: string; } // UNIQUE(memberId, kind, refId) → INSERT ... ON CONFLICT DO NOTHING (no double-award)

3 · Level rule
// pure, idempotent, recomputable from points alone const TIERS = [ { level: 1, name: 'Apprentice', min: 0 }, { level: 2, name: 'Builder', min: 1500 }, { level: 3, name: 'Architect', min: 2700 }, { level: 4, name: 'Principal Architect', min: 4200 }, ]; const levelFor = (pts: number) => TIERS.reduce((a,t) => pts >= t.min ? t : a, TIERS[0]); const progressToNext = (pts: number) => { /* (pts - cur.min) / (next.min - cur.min); 100% at max tier */ };
