9, 10 — Hooks

check-emdash.sh (PostToolUse)

Fires on Edit / Write / MultiEdit. Extracts file_path from the JSON hook payload (no jq dependency, uses grep/sed). Only acts on backend/src/scripts/send*.js|ts|mjs|cjs. Counts em-dash byte sequences (U+2014 = 0xE2 0x80 0x94). If > 0, exits 2 with the offending line numbers printed to stderr — Claude sees that and self-corrects.

Tested 3 ways:

  • Clean file (no em-dashes) → exit 0 ✓
  • File with 1 em-dash → exit 2, prints line number + content ✓
  • Non-send-script path → skipped, exit 0 ✓

session-end-progress-audit.sh (Stop)

Fires when the session ends. Counts PROGRESS.md entries dated today, lists git-modified files in gated paths (backend/frontend/scripts/nginx/directives), warns loudly if there are gated-path modifications but zero PROGRESS.md entries for today. Also flags if the last commit touched gated paths without touching PROGRESS.md. Informational only (exit 0 always) — Stop hooks that block exit are disruptive; this is awareness, not enforcement.

Both hooks registered in .claude/settings.json under hooks.PostToolUse and hooks.Stop.

11, 12 — MCP servers

colaberry-portal-api (7 tools)

get_system_state, explain_task, get_telemetry, get_telemetry_health, get_state_graph, get_database_map, get_ui_map. Read-only. Auth via PORTAL_BEARER_TOKEN env var if set. Truncates responses at 50KB to prevent context bombs.

Smoke-tested: tools/list request returns the 7 tools with full descriptions and input schemas.

colaberry-postgres-analytics (6 tools)

5 named tools (query_visitor_sessions_by_site, query_leads_by_source, query_recent_pageviews, query_campaign_health, query_visitor_funnel) + 1 escape hatch (run_safe_query) for ad-hoc SELECTs.

Hard safety: every query passes through isReadOnly() which rejects any statement containing INSERT|UPDATE|DELETE|DROP|TRUNCATE|ALTER|CREATE|GRANT|REVOKE|COMMIT|ROLLBACK|VACUUM|REINDEX|COPY|LOCK AND requires the statement to start with SELECT|WITH|SHOW|EXPLAIN. Two-gate: deny-list AND allow-list. Connection: ssh root@95.216.199.47 docker exec accelerator-db psql ..., no persistent connection.

Smoke-tested: tools/list request returns the 6 tools with descriptions.

Both servers registered in .claude/settings.json under mcpServers. They start automatically on Claude Code session start; communicate via stdio.

13 — Plugin packaging

.claude/plugin.json bundles every asset for distribution: settings, claudeignore, 7 CLAUDE.md files (1 root + 6 subdir), 8 skills (5 design + 3 new), 2 hooks, 2 MCP servers. Lists requirements (Claude Code >= 1.0, Node >= 18, 5 npm devDependencies), names the DRI, and sets next review cadence.
