2.2 Created .claude/settings.json with team-wide permissions

Project-level settings committed to the repo. Every developer (Ali, Tejesh, Ram, future hires) and every Claude session gets the same baseline.

Deny list blocks the genuinely-dangerous patterns regardless of who's running:

  • rm -rf /*, rm -rf ~, rm -rf ~/*
  • git push --force *, git push -f *
  • git reset --hard origin/main*, git branch -D main*
  • DROP TABLE*, DROP DATABASE*, TRUNCATE TABLE*
  • Any command containing --no-verify or --no-gpg-sign (defeats commit hooks/signing)

Allow list is the safe daily-driver baseline: read commands, npm/node, docker exec, ssh to prod (read scope), curl, git status/diff/log/add/commit, gh, npx tsc/jest/playwright. Excludes any tokens, JWTs, or one-off paste commands.

A .claude/settings.json (new, ~110 lines, ~80 allow rules, ~13 deny rules)
