## Primary persona: corporate executive driving AI adoption

The primary user persona for the accelerator is the corporate executive tasked
with steering their organization toward AI adoption. The persona spans CEOs,
CTOs, CIOs, and other senior leaders responsible for strategic decision-making.

| Attribute | Value |
| --- | --- |
| Demographics | Aged 35–60, often holding advanced degrees in business, technology, or related fields |
| Goals | Enhance operational efficiency; drive innovation; maintain competitive edge |
| Pain points | Technical aspects of AI; lack of skilled personnel; integrating new tech into existing workflows |
| Behavior patterns | Prefer concise information, actionable insights, strategic overviews over deep technical detail; data-driven; appreciate visual representations of complex data |

### Example: Mark Johnson, CEO of Tech Innovations Inc.

| Attribute | Value |
| --- | --- |
| Background | Over 20 years of experience in technology leadership |
| Goals | Implement AI to streamline operations and improve customer experience |
| Pain points | Limited understanding of AI applications; concerns about data security |

Mark is the concrete illustration of the primary persona — used in copy,
demos, and onboarding so the abstract "executive driving AI adoption"
profile has a face and a name reviewers can reason against. This is the
artifact that satisfies requirement **REQ-048**.

## Secondary personas

The secondary personas are stakeholders who interact with the accelerator
alongside the primary executive.

### IT managers

| Attribute | Value |
| --- | --- |
| Demographics | Aged 30–50, backgrounds in IT or computer science |
| Goals | Ensure technology integrates seamlessly and meets security compliance |
| Pain points | Resource limitations; balancing multiple projects |

### Data scientists

| Attribute | Value |
| --- | --- |
| Demographics | Aged 25–45, degrees in statistics, mathematics, or computer science |
| Goals | Access to data and tools to develop predictive models and insights |
| Pain points | Need for high-quality data and robust computational resources |

### Corporate trainers

| Attribute | Value |
| --- | --- |
| Demographics | Experienced training-and-development professionals, aged 30–50 |
| Goals | Develop effective training materials and programs |
| Pain points | Limited resources for creating engaging content |

## How to use this artifact

- Marketing + sales reference Mark Johnson when illustrating the primary ICP.
- The semantic verifier can match REQ-048 against this file by name and content.
- When new personas surface in customer-discovery work, extend this file
  rather than scattering definitions across the codebase.
