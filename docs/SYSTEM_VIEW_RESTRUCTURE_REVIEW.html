01 What shipped (one paragraph)

One new lean page. Two route changes. One legacy banner.

A new frontend/src/pages/project/SystemView.tsx ships as a focused ~300-line page with 5 tabs: Components (default), Architecture, BPs, Operations (advanced), Cognition (advanced). CORE tabs load eagerly; ADVANCED tabs lazy-mount on first click. Tab persistence via ?tab=... URL parameter. Each tab is a thin wrapper around an existing component (CapabilityGrid, SystemArchitectureCard, PortalBusinessProcessesTab, AutonomousExecutionDashboard, OperatorCognitionDashboard) — no component logic was rewritten. Both advanced dashboards use the existing defaultCollapsed={true} prop so users see a calm summary, not a dump. Route /portal/project/system-v2 now serves SystemView; the legacy 4,295-line SystemViewV2 moves to /portal/project/system-v2-legacy with a top-of-page warning banner. Backend untouched. Frontend tsc --noEmit exit 0.

"System View no longer competes with Cory. It explains; Cory decides."

03 The 5 tabs

Each tab has one job. Advanced tabs are visually marked.
#TabTierPurposeMounts
1 Components (default) CORE What exists in the system CapabilityGrid
2 Architecture CORE How the system is connected SystemArchitectureCard + a hint pointing to BPs tab for per-BP visualization
3 BPs CORE Business process structure PortalBusinessProcessesTab (which mounts BP detail panel + per-BP visualization on selection)
4 Operations advanced ADVANCED Runtime visibility (operator dashboard) AutonomousExecutionDashboard defaultCollapsed=true + intro disclaimer
5 Cognition advanced ADVANCED Intelligence visibility (reasoning continuity) OperatorCognitionDashboard defaultCollapsed=true + intro disclaimer

Lazy-mount behavior

  • Components tab mounts on initial render (it's the default).
  • Architecture / BPs / Operations / Cognition mount the first time the user clicks them.
  • Once mounted, a tab stays mounted (preserves scroll + state) but is hidden when another tab is active.
  • This means: a user who never clicks "Cognition" never pays its data-fetch cost.

05 Components tab

The default tab — what most users see when they land on System.

What it contains

  • CapabilityGrid mounted inline.
  • Project completion summary card (overall %).
  • Capability cards: name, description, completion %, feature count, status, scope toggle, optional AI badge.
  • Click a card → expands CapabilityDetail drawer.
  • "Add Feature with AI" → AIFeatureBuilder modal.

What it does NOT contain

  • "Build" buttons that bypass Blueprint. Scope toggle is configuration, not execution.
  • "Next step" recommendations — those live at Home.
  • Priority rankings — Cory's authority.

Why the existing CapabilityGrid is fine without modification: it already only exposes scope management (toggle active/inactive) and feature creation — both are configuration, not execution. Per the surface contract, configuration in System is allowed; only execution is forbidden.

Critique & notes — section 5

06 Architecture tab

How the layers fit together.

What it contains

  • SystemArchitectureCard — backend / frontend / agents / data layer view.
  • A friendly hint pointing to the BPs tab for per-BP visualizations.

What it does NOT contain

  • Per-BP visualizations bare. ProcessVisualPanel requires a BP context (links + usability props), so it only makes sense from inside the BPs tab when a process is selected.
  • "Refresh architecture" or "Edit architecture" buttons — System explains, Cory decides changes.

An info strip at the bottom of the tab tells the user: "For a per-BP visual flow, open the BPs tab and pick a process." This avoids the mistake of showing a context-less visualization when the user really wants per-BP detail.

Critique & notes — section 6

07 BPs tab

Process structure, status, dependencies.

What it contains

  • PortalBusinessProcessesTab mounted inline.
  • BP list with lifecycle / layer filters, completion %, total + matched requirements, by-layer health badges.
  • Click a BP → PortalBusinessProcessDetail opens with the per-BP visualization (this is where ProcessVisualPanel lives, scoped properly to the selected BP).

What it does NOT contain

  • "Run BP" or "Execute BP" buttons — execution flows through Blueprint via Cory.
  • Priority sorting that competes with Cory's queue.

The existing PortalBusinessProcessesTab imports SteeringPanel; that's already marked subordinate-to-Cory in its file header, and steering hints route through the unified state.

Critique & notes — section 7

08 Operations tab (advanced)

Runtime visibility — opt-in, not in your face.

What it contains

  • Intro card explaining: "You don't need to read these to use the platform."
  • AutonomousExecutionDashboard defaultCollapsed=true — every phase section starts collapsed, expanded individually.

How progressive disclosure works

  • Tab is lazy-mounted (no fetch until first click).
  • Once mounted, dashboard sections (Phase 13–32) all start collapsed.
  • Operator expands only what they want to inspect.
  • Default visual: a summary line + chevron per section.

Visual treatment

The advanced pill on the tab strip + intro card make it visually clear this is secondary. Color treatment uses purple accents (consistent with the L4 / advanced visual identity established in earlier sprints).

Critique & notes — section 8

09 Cognition tab (advanced)

Intelligence visibility — same disclosure rules as Operations.

What it contains

  • Intro card framing: "How the platform reasons about itself over time. Hidden infrastructure — surfaced for transparency, not because you need to act on it."
  • OperatorCognitionDashboard defaultCollapsed=true.

Why this exists separately from Operations

Operations is about what's happening right now (active autonomy lanes, recent governance actions). Cognition is about how the platform thinks over time (governance memory, reasoning continuity, handoff archaeology, federation state). Two distinct conceptual spaces — keeping them as separate tabs lets a power user navigate to the one they want without scrolling past the other.

13 What this sprint did NOT do (intentional)

Honest scoping.

Out of scope this sprint

  • Did NOT delete SystemViewV2.tsx. The legacy surface is preserved at /system-v2-legacy with a warning banner.
  • Did NOT migrate ProjectSetupWizard out of the legacy. That move belongs in the Setup Route Extraction sprint (next).

  • Did NOT touch the backend.
  • Did NOT add new dashboards or visualizations.
  • Did NOT extract the operator dashboards into multiple smaller pieces — they remain large monoliths, just now hidden behind progressive disclosure.
