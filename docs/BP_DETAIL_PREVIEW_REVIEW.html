3Which BPs get a preview — and the honest data gap

The preview renders when the BP has a real frontend_route. Auditing production showed two kinds of "page" BP:

Kindsourcefrontend_routeGets a preview?Examples
Route-tree pages frontend_page set (/program, /pricing, /enroll, /contact, /sponsorship) YES Program Page, Pricing Page, Enroll Page, Contact Page, Sponsorship Page
Brownfield-discovered brownfield_discovered null NO (correctly) AI Architect Landing Page, AI Workforce Designer Page, Agency Partner Page, Alumni Champion Page, Advisory Page, Case Studies Page
Why the brownfield BPs don't preview — and why that's correct behavior:
  • ~34 BPs are named like pages but were discovered by the brownfield requirement-extraction process from documents — not from the actual route tree. They carry no frontend_route.
  • The component does the right thing: it only embeds an iframe when there's a real route to embed. No route → no preview section → the modal renders exactly as it did before.
  • The real fix is upstream: the brownfield discovery should link these named-page BPs to their actual routes (e.g. "AI Architect Landing Page" → some /ai-architect route that exists in the codebase). That's a data-quality task for the discovery engine, not the detail modal.
