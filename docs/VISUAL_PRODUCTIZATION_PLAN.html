01 Why we're pivoting

The backend cognition substrate is sufficiently advanced. The next bottleneck is human confidence, not system correctness.
"Is it truly a much better system — something I can take to the market?" Honest answer: the architecture is excellent enterprise plumbing, but the user-facing surface has not kept pace. Stop adding governance layers; pivot to product.

What we have

  • 32 backend phases shipped — 1,732 tests passing across 32 suites, zero regressions.
  • Hard architectural vetoes hold — cross-org isolation, no operator profiling, no autonomous mutation outside Phase 27/28 envelope+quota lane, deterministic replay everywhere.
  • ~150+ frontend components — many overlapping, several over the 500-line ceiling, multiple competing "next action" surfaces.

What's blocking market readiness

  • The user lands on a page and cannot tell what to do next without explanation.
  • Multiple surfaces show different "priority" rankings for the same project.
  • "Build" buttons appear in 6+ places, each with subtly different prompt assembly.
  • Advanced cognition (archaeology, topology, federation, replay) is rendered as foreground UI — intimidating, not reassuring.
  • No fast loop for visual critique → structured prompt → rebuild.

03 Unified UX philosophy

Eight principles. Every product decision must trace to one of these. If a screen, a button, or a metric doesn't, cut it.
PRINCIPLE 01

Calm UI

Reduce clutter, competing actions, duplicate navigation, fragmented flows. Premium feel: deterministic, trustworthy, operationally stable.

PRINCIPLE 02

One execution lane

The user never builds from cards, groups, or random tabs. All execution routes through Build Center: prompt prep + context assembly + validation + execution coordination.

PRINCIPLE 03

One next step

Cory exposes one prioritized operational queue. If a BP is next, it must be first everywhere. Every surface agrees. No competing rankings.

PRINCIPLE 04

One calculation system

UnifiedProjectState is the authoritative state object. All readiness, coverage, confidence, health, priority derives from it. No local recalculation, ever.

PRINCIPLE 05

Consistent navigation

Every BP, component, and page uses the same six tabs: Summary, Tasks, Build, Verification, Intelligence, History. No variation.

PRINCIPLE 06

Hide advanced complexity

Archaeology, topology, federation, governance replay, distributed runtime, stabilization — all become background intelligence, not foreground UX. Invisible but powerful.

PRINCIPLE 07

Critique → prompt

The Visual Engineering Workspace is a prompt compiler, not a review page. Critique structures requests; the workspace generates Claude Code prompts the user pastes and runs.

PRINCIPLE 08

Synchronized intelligence

Structured registry files (system map, DB map, route registry, BP registry, component registry) are the canonical truth and stay in sync via build manifests — no more drift.

What "Compile prompt" does

  1. Reads the target file path from the page route registry (system intelligence file).
  2. Pulls the relevant component source via the component registry.
  3. Bundles the pinned critiques + region hints + suggested edits into a single Claude Code prompt.
  4. Adds verification scaffolding (ask Claude to run tsc --noEmit, screenshot the result).
  5. Opens the prompt in a new tab with one-click copy. User pastes into Claude Code; rebuild starts immediately.

Why this is the acceleration engine

Today's loop: see issue → describe in chat → explain location → explain context → wait for clarifying questions → rebuild. ~10–25 minutes per change.

Visual Workspace loop: see issue → click → type → group → compile → paste → rebuild. ~2–4 minutes per change. 5–6× speedup on QA cycles.

10 UnifiedProjectState — one calculation system

The single authoritative state object for a project. All UI calculation derives from this; nothing recalculates locally.

Shape (sketch)

interface UnifiedProjectState {
  project_id: string;
  display_name: string;
  active_bp_count: number;

  // Cory's queue — ONE prioritized list
  next_action: NextAction;
  queue: QueueEntry[];

  // Calculations — derived once, consumed everywhere
  readiness: { score: 0..100; band: 'red' | 'amber' | 'green'; reasons: string[] };
  coverage: { score: 0..100; bps_covered: number; bps_total: number };
  health: { score: 0..100; regressions_24h: number; verification_pass_rate: number };
  confidence: { score: 0..100; sources: string[] };

  // References to background intelligence — available, not foregrounded
  intelligence_drawer_refs: {
    operations_summary_url: string;
    cognition_summary_url: string;
    governance_summary_url: string;
  };

  built_at: string;
}

Rules

  • One backend endpoint produces it (GET /api/portal/project/:id/unified-state).
  • One frontend hook consumes it (useUnifiedProjectState(projectId)).
  • Components display values; they never recompute. Lint rule: any local readinessScore = in a component is a violation.
  • Backend derivation re-uses existing engines (SystemStateEngine, BP planner) as inputs — this is the synthesizer, not a replacement.

11 Critique → prompt workflow

The full loop, end to end. This is the operating tempo we're optimizing for.
  1. Open a page. User picks any page in the platform from Visual Workspace sidebar.
  2. Page renders embedded. Iframe loads the live URL with annotation overlay enabled.
  3. User clicks regions. Each click drops a numbered pin and opens a critique input.
  4. Optional: AI suggests. "AI suggest" reads the pinned region's source HTML and proposes a refined edit.
  5. User groups pins. Related pins get grouped into a single change request (e.g. "Header refinement").
  6. Compile. "Compile prompt" assembles a Claude Code prompt: file paths (from registry), pinned critiques, suggested edits, verification scaffolding.
  7. Open / copy. Prompt opens in a new tab with one-click copy. User pastes into Claude Code.
  8. Claude Code rebuilds. Files modified, manifest emitted, verification kicked off.
  9. Manifest received. Verification Workspace auto-reruns relevant checks; UnifiedProjectState refreshes.
  10. Visual workspace re-renders. User reloads the iframe; pins resolve automatically when source matches the requested edit.

Per-cycle target: 2–4 minutes from spotted issue to verified rebuild. Today's equivalent loop: 10–25 minutes.

16 Out of scope & explicit non-goals

Listed so future Claude doesn't drift back into governance phase territory.

Strictly out of scope for this productization arc

  • Phase 33 or any subsequent governance phase. Backend is mature enough.
  • New cognition subsystems. Archaeology / topology / federation / replay are background-only from now on.
  • Removing existing governance. Phases 1–32 stay; they just stop owning UI surface.
  • Renaming Cory. Cory is the brand of operational authority; no rebrand.
  • Multi-platform UI rewrite. Stay on React+Bootstrap; no migration to a new framework.
  • Mobile-native app. Web responsive only.
  • "AI agent marketplace." Not yet; product clarity first.
  • Replacing PROGRESS.md. It works; keep it as the human log.
