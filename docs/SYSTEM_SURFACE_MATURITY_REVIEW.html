2Progressive reveal — collapsed → expanded → inventory

State A — collapsed (default)

5 domain rows visible above the fold · each row carries: icon · name · narrative line · completion % · tone word · chevron-down · One scan, full operational picture

State B — one domain expanded

Click "Intake & Registration" → reveals one compact BP row underneath: name · req count · tone word · chevron-right · click → BP detail modal · No metric-bar grids · No "monitoring widget" energy

State C — full inventory (escape hatch for power users)

"Show full inventory" toggle reveals the legacy 3-column metric-bar grid · operator can still inspect every BP at once if they need to · but it's no longer the default budget

  • ≥80% → "healthy" tone: "Reporting systems have high confidence — N processes, X% complete."
  • 40–79% → "partial" tone: "Lead orchestration is partially connected — N processes, X% complete."
  • <40% → "early-stage" tone: "Marketing operations need foundational work — N processes, X% complete."
